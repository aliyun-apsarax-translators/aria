在一些术语已经在所在位置被定义的同时，下列定义会适用于整份文档。

无障碍 API

操作系统和其它平台提供了一套给辅助技术暴露对象和事件信息的接口。辅助技术使用这些接口来获取相关信息并和那些挂件进行交互。无障碍 API 的例子有 微软主动式无障碍 [[MSAA]]、微软用户接口自动化 [[UI-AUTOMATION]]、MSAA 和 UIA 直达 [[UIA-EXPRESS]]、 Mac OS X 无障碍协议 [[AXAPI]]、Linux/Unix 无障碍工具集 [[ATK]] 和 辅助技术服务提供者接口 [[AT-SPI]] 以及 IAccessible2 [[IAccessible2]]。

无障碍子树

无障碍树中的一个无障碍对象以及它在那棵树中的后代。它不包含那棵树中父-子关系之外的对象。例如不包含通过 aria-flowto 链接起来的那些对象，除非那些对象同时也是这棵无障碍树的后代。

无障碍树

无障碍对象的树，表现了用户界面 (UI) 的结构。无障碍树里的每个结点都表现了一个 UI 中的元素，并通过无障碍 API 暴露出来；例如一个按钮、一个复选框或容器。

无障碍描述

一段提供了额外信息的无障碍描述，关联到一个界面上的元素，作为无障碍名称的补充。无障碍描述可以是视觉上可接收的，也可以不是。

无障碍名称

无障碍名称是一个用户界面元素的名称。每个平台的无障碍 API 都提供了无障碍名称的 property。无障碍名称的值可以源自用户界面元素上可见的 (例如可见的文本或按钮) 或不可见的 (例如描述一个图标的文本替代品) property。请查阅无障碍描述。

无障碍名称 property 的一个简单的用法可能是通过一个 "OK" 按钮来说明。它的文本 "OK" 是无障碍名称。当按钮接收到焦点时，辅助技术可以把平台的角色描述和无障碍名称衔接起来。例如一个读屏软件可以念作 "电钮 OK" 或 "OK 按钮"。衔接的顺序和具体的角色描述 (例如 "按钮"、"电钮"、"可点击的按钮") 由平台的无障碍 API 或辅助技术来决定。

无障碍对象

一个平台的无障碍 API 的无障碍树中的一个结点。无障碍对象暴露了若干个state、property和事件为辅助技术所用。一般在标记语言、尤其是 WAI-ARIA 的上下文中 (例如 HTML 和 SVG)，标记元素及其特性会被展示为无障碍对象。

Activation behavior

The action taken when an event, typically initiated by users through an input device, causes an element to fulfill a defined role. The role may be defined for that element by the host language, or by author-defined variables, or both. The role for any given element may be a generic action, or may be unique to that element. For example, the activation behavior of an HTML or SVG <a> element shall be to cause the user agent to traverse the link specified in the href attribute, with the further optional parameter of specifying the browsing context for the traversal (such as the current window or tab, a named window, or a new window); the activation behavior of an HTML <input> element with the type attribute value submit shall be to send the values of the form elements to an author-defined IRI by the author-defined HTTP method.

辅助技术

硬件或软件满足：

  • 依赖一个用户代理提供的服务来获取和渲染 web 内容
  • 通过使用 API 和一个用户代理或 web 内容本身工作在一起，以及
  • 超越用户代理提供的东西来提供服务，促进有障碍的用户和 web 内容互动

这个定义可能不同于其它文档。

在文档上下文中非常重要的辅助技术的例子有：

  • 屏幕放大器，用来放大并改善被渲染文本和图片的视觉可读性；
  • 屏幕阅读器，经常用来把信息转换成为合成语音或可刷新的盲文展示；
  • 文本转语音软件，用来把文本转换成为合成语音；
  • 语音识别软件，允许语音控制和叙述；
  • 替代输入的技术 (包括头动指针、屏幕上的虚拟键盘、单项开关和呼气/吸气控制设备)，用来模拟键盘；
  • 替代指针的设备，用来模拟鼠标和点击。
Attribute

在这份规范里，attribute 和其在标记语言中的用法一样。Attribute 是添加在元素上的结构化的特性，提供该元素展示的对象的 state 和 property。

类

一组共享相同特征的实例对象。

已废弃

一个已废弃的角色、state 或 property是已经因构造了更新的或情况发生变化而陈旧的，并在将来的版本中可能被移除的 WAI-ARIA 规范。用户代理被鼓励继续支持被定位已废弃的项目，以获得更好的向后兼容性。更多信息参见一致性章节的已废弃的要求。

桌面聚焦事件

通过无障碍 API 来/到宿主操作系统的事件，提示输入焦点的改变。

元素

在这份规范里，元素和其在标记语言中的用法一样。元素是为对象包含数据情况的结构化的标记语言元素。

事件

一个程序化的信息，用于计算系统中沟通不相关的对象的 state 的改变。通过抽象的事件来描述用户在网页上的输入是常见的，它描述了交互并提供文档对象 state 改变的通知。在一些编程语言中，事件是通过消息提醒被认知的。

暴露

根据WAI-ARIA 用户代理实现指南的定义翻译到平台相关的无障碍 APIs。[[WAI-ARIA-IMPLEMENTATION]]

图形化文档

一个包含了图形化展示的用户可导航的文档。图表、地图、示意图、蓝图和仪表盘都是图形化文档的例子。一个图形化的文档会使用任何符号、图片、文字和原始的图形 (如圆、点、线、路径、矩形等图形) 来组合而成。

隐藏

表示这个元素对任何用户来说都是不可见的、不可察觉的、也不可交互的。一个隐藏元素是指它或它的上层祖先元素没有被渲染或明确被隐藏。

提供有用信息的

提供信息且不要求一致性的内容。要求一致性的内容会被认为是规范的。

键盘无障碍

对使用键盘或诸如可呼气/吸气管道的模拟键盘输入的辅助技术的用户无障碍。这份文档中提到的内容关联到了WCAG 2.0 指南 2.1: 使所有功能都能通过键盘来操作 [[WCAG20]]。

地标

网页上用户可以快速访问到的一类区域。网页里这样的不同区域里的内容是不同的，同时和具体的用户目的相关联，诸如导航、搜索、详读主题内容等。

活区域

活区域是一个网页上的可察觉区域且通常会在用户聚焦到别的地方时通过别的事件产生变化。这些区域的变化不总是由用户交互引起。这类实践随着使用 Ajax 的增多而变得普遍。活区域的例子包括一个聊天记录、股票收报或一个随时更新的反应赛事统计的计分牌。因为这些不同的地方都可能在用户焦点区域之外发生改变，诸如屏幕阅读器等辅助技术不会注意到它们的存在或无法告知用户。WAI-ARIA 已经提供了一个 property 集合来允许作者标识活区域并处理它们：aria-live、aria-relevant、aria-atomic 和 aria-busy。

主要内容元素

一个实现宿主语言的主要内容元素，例如 HTML 中的 body 元素。

被管理的 state

被用户代理控制的无障碍 API 的 state，例如焦点和选择。它们会和被作者控制的"不被管理的 state"放在一起考虑。尽管如此，作者可以覆写一些被管理的 state，例如 aria-posinset 和 aria-setsize。很多被管理的 state 有相应的例如 :focus 的 CSS 伪类和例如 ::selection 的 CSS 伪元素，它们同样会被用户代理更新。

内梅斯布莱叶

内梅斯布莱叶数学码 (Nemeth Braille Code for Mathematics) 是一种为数学和科学符号编码的盲文。查阅维基百科上的 Nemeth Braille了解更多。

结点

DOM 树或无障碍树中的对象的基本类型。由 DOM 结点再指定，就包括了元素或文本结点。一个无障碍树上的结点是无障碍对象。

规范的

有一致性的要求。相对来说，被认为提供有用信息的或“不规范的”内容是不要求一致性的。

对象

在用户界面的上下文中，是一个具有可感知用户体验，在标记语言中以一个或多个元素呈现，且被用户代理渲染的项目。

在编程的上下文中，是一个或多个类和定义类似对象一般化特征的接口的实例化。一个无障碍 API 可能体现了一个或多个 DOM 对象。无障碍 API 定义了区别于 DOM 接口的接口。

本体

一段类的特征的描述以及它们之间如何相关联。

可操作性

可以以用户可以控制的方式使用。这份文档中的引用关联了 WCAG 2.0 原则2：可操作性-内容必须可操作 [[WCAG20]]。更多可查阅键盘无障碍。

被拥有的元素

一个“被拥有的元素”是这个element的任何 DOM 后代、任何通过 aria-owns 指定为子元素的元素、或这个被拥有的子元素的任何 DOM 后代。

拥有元素

一个“拥有元素”是这个元素的任一 DOM 的祖先、或任何带有引用这个元素的一个 aria-owns attribute 的元素。

可感知

可以以用户可以感受到方式展示出来。这份文档中的引用关联了 WCAG 2.0 原则1：可感知性-内容必须可感知 [[WCAG20]]。

Property

一个给定对象所必需的 attribute，或代表一个和对象相关联的数据值。一个 property 的改变可能会现住地影响一个对象的意义或展示。确定的 property (例如 aria-multiline) 比 state 更稳定，更不会发生改变，但是注意变更的频度并不是一条规则。一些 property，诸如 aria-activedescendant、aria-valuenow 和 aria-valuetext 也是经常改变的。查阅对 state 和 property 的澄清了解更多。

联系

两个不同事物之间的连接。联系表示方式的可以是多种类型的，例如一个对象标记了或控制了另一个对象等。

角色

主要的标识类型。这个语义关系允许工具展示并支持和对象的交互方式，以符合用户对同类对象的预期。

根 WAI-ARIA 结点

包含了非元数据的内容的主要元素。在许多语言中，这就是文档元素，但是在 HTML 中是 <body>。

语义

一些对人类来说可以理解的意思，以计算机可以处理的对象展示的方式被定义，例如元素和 attribute，并以各类人都可以相互理解的对象的方式可靠地展示对象。

State

一个 state 是表示一个对象特征的动态的 property，可能会由于用户交互或自动化流程而改变。State 不会影响对象的必然本质，但是会展示关系到对象或用户交互可能性的数据。查阅对 state 和 property 的澄清了解更多。

子文档

任何通过 <frame>、<iframe> 或类似机制创建出来的文档。一个子文档可能包含一个文档、一个应用或任何挂件，诸如从另一个服务器拉来的日历。在无障碍树中，有两种这样情况的无障碍对象——一个代表了父级文档中的 <frame>/<iframe> 元素，该父级文档是一个无障碍对象子级内容，代表了衍生出来的文档内容。

目标元素

一个在 WAI-ARIA 关系描述中被指定的元素。例如 <div aria-controls="elem1"> 中的 "elem1" 是这个目标元素的 ID。

分类系统

一个对于多种类之间关系的分级的定义。分类系统中的类的属性继承自它的超类。一个分类系统可以构成一部分本体的正式定义。

文本结点

DOM 结点的一种，代表一个 attribute 或元素的文本内容。文本结点没有任何子结点。

工具提示 attribute

会让一个用户代理生成工具提示的任何宿主语言的 attribute，例如桌面端用户代理中对鼠标悬停的响应。

可理解性

可以展示给用户并使其产生结构化含义。这份文档中的引用关联了 WCAG 2.0 原则3：信息和用户界面操作必须是可理解的 [[WCAG20]]。

用户代理

任何获取、渲染、促使最终用户和 web 内容交互的软件。这个定义可能和其它文档中的定义有所不同。

有效的 IDREF

一个匹配同一个文档中目标元素 ID 的引用。

挂件

用户可交互的分离的用户界面对象。挂件既包含了只有一个值或操作的简单的对象 (如复选框或菜单项)，也包含了管理很多子对象的复杂对象 (例如树或网格)。
